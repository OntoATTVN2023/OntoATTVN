Definition

CVE, or Common Vulnerabilities and Exposures, is a dictionary of common names for publicly known information security vulnerabilities.

1
Function

CVE Identifiers (CVE ID) are unique, common identifiers used for assigning vulnerabilities in a variety of both commercial and open-source software, such as including operating systems, web servers, databases, and applications.

2
Organization

They are assigned by the CVE Program, which is managed by the MITRE Corporation since 1999.

3
Definition

CWE, or Common Weakness Enumeration, is a community-developed list of common software and hardware weakness types that have security ramifications.

1
Classify

CWE is a hierarchical taxonomy, with each weakness being assigned a unique identifier and a set of descriptive attributes. The CWE taxonomy is divided into three categories:

  • Hardware Design
  • Software Development
  • Research Concepts
2
Definition

CAPEC, or Common Attack Pattern Enumeration and Classification, as the name suggests, is a catalogue of common attack patterns frequently used by threat actors to exploit vulnerabilities in software and other cyber-enabled capabilities.

1
Classify

Like CWE, CAPEC is arranged in a similar hierarchical taxonomy: attack patterns are ordered into categories and sub-categories, with each pattern being assigned a unique identifier and a set of descriptive attributes. The CAPEC taxonomy is organized into two main categories:

  • Mechanisms of Attack
  • Domains of Attack
2
Definition

MITRE ATT&CK, stands for Adversarial Tactics, Techniques, and Common Knowledge is a globally-accessible knowledge base of adversary tactics and techniques based on real-world observations.

1
Benefit

The process MITRE used to create ATT&CK, and the philosophy that was developed and used for curating new content, are critical aspects of the work and are very useful for other efforts that strive to create similar adversary models and information repositories.

2
Objectives

The MITRE ATT&CK matrix contains a set of techniques used by adversaries to accomplish a specific objective. Those objectives are categorized as "tactics" in the ATT&CK Matrix. The objectives are presented linearly from the point of reconnaissance to the final goal of exfiltration or "impact".

3

In the field of cyber security, ontology can be used to contextualize weaknesses and attack patterns using entity-relationship framework. The result after which can be utilised to create an intuitive knowledge pool, offering a systematic view of vulnerabilities, attacks, and other security-related aspects.

1

How does it work?

  • Search your CVE ID

    Find the latest CVEs within the National Vulnerability Database via our search box.

  • Map to Ontology

    After choosing your CVE ID, our application will inspect and assign the vulnerability to its appropriate category within ATT&CK matrices through an analysis chain of CWE, CAPEC.

  • Show the output

    The result of our analysis will be displayed with a list of CWE associated with the CVE, consisting of Technique, Sub-Technique, Tactics used, and a graph displaying what Stage this CVE most likely to be used.

ATTVN is a group of 5 students who developed this software as a part of our final capstone project. ATTVN provides an online lookup platform and helps convert from a CVE vulnerability to the techniques, tactics and strategies of cyber attackers according to the MITRE ATT&CK framework to help users easily grasp the stages of an attack through a specified vulnerability.

This application is open-sourced, which means anyone can access and contribute to the source code.
